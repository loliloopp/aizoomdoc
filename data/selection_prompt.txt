Твоя задача — найти в тексте ИЗОБРАЖЕНИЯ, необходимые для ответа на запрос пользователя.

ВАЖНО ПРО СТРУКТУРУ ДОКУМЕНТА:
1. Документ содержит блоки описания изображений в формате JSON внутри ```json ... ```.
2. Каждый блок содержит:
   - `doc_metadata`: метаданные (имя файла, номер страницы).
   - `image`: объект с полем `uri` — ПРЯМАЯ ССЫЛКА на PDF-файл изображения.
   - `raw_pdfplumber_text`: основной текст со страницы.
   - `analysis`: объект с результатами анализа, содержащий вложенный объект `analysis`:
     - `content_summary`: краткое описание содержимого.
     - `detailed_description`: подробное описание графики.
     - `clean_ocr_text`: распознанный текст (OCR) на изображении.
     - `key_entities`: найденные ключевые сущности (маркировки, размеры).

ИНСТРУКЦИЯ:
1. Прочитай запрос пользователя.
2. Найди в тексте блоки JSON, которые релевантны запросу.
   - Ищи ключевые слова в `analysis.analysis.content_summary`, `analysis.analysis.detailed_description` и `analysis.analysis.clean_ocr_text`.
   - `raw_pdfplumber_text` также может содержать полезный контекст.
3. Извлеки `image_id` (если есть) или URL изображения из поля `image.uri` внутри найденного JSON блока.
4. Верни JSON:
```json
{
  "reasoning": "Нужен план 1 этажа (найден блок на стр. 9 с описанием 'Ситуационный план')",
  "needs_images": true,
  "image_urls": ["https://..."]
}
```
Если картинок нет или они не нужны - верни `needs_images: false`.
